Infrastructure as code

Azure Terraform

This lesson introduces a Terraform workflow for creating Azure resources from version-controlled code.

Terraform files to practice

  • providers.tf: configure Terraform, AzureRM provider, and provider versions.
  • resource-group.tf: create the Azure resource group and shared tags.
  • network.tf: create the VNet and subnet.
  • security.tf: attach a network security group for SSH practice.
  • storage.tf: create a storage account with secure defaults.
  • vm.tf: create public IP, NIC, and Linux VM resources.

Practice workflow

  • Run terraform fmt to keep files consistent.
  • Run terraform validate before opening a pull request.
  • Use terraform plan to review resource group, VNet, subnet, VM, and storage changes.
  • Use a service connection or federated credentials for Azure DevOps authentication.
  • Keep state in a remote backend for team practice projects.
